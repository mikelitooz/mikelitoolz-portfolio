You are designing a developer portfolio website.
Your goal is to establish a cohesive, modern, and responsive typography system that feels both stylish and professional.
The fonts used are:

Space Grotesk for all headings

Inter for all body text

Typography should communicate confidence, clarity, and precision — reflecting the personality of a skilled developer who builds sleek, high-quality web applications.

🎯 Core Objectives

Maintain a clear visual hierarchy across all pages.

Keep typography consistent, balanced, and responsive at every breakpoint.

Ensure spacing, font weight, and scale remain harmonious throughout the design.

Avoid visual clutter — simplicity and legibility take priority.

🔠 Hierarchy & Roles

Primary Headings (H1): Use Space Grotesk with bold weight. These should be strong and attention-grabbing, used mainly for the hero section and major page titles.

Secondary Headings (H2–H3): Use Space Grotesk with medium-to-bold weight, creating visual rhythm between sections without overwhelming.

Body Text (P): Use Inter for paragraphs and long-form content. It should feel relaxed, readable, and perfectly spaced for comfort.

Supporting Text (small, captions, labels): Use lighter weights of Inter, slightly reduced in size and contrast to provide hierarchy without distraction.

Maintain clear size relationships between heading levels — each step down should feel natural and proportional, never abrupt.

📏 Spacing & Readability

Keep ample white space around headings and paragraphs; text should breathe, not feel cramped.

Maintain consistent vertical spacing across all sections (e.g., equal top and bottom margins for headings).

Use line height that supports clarity — generous for body text, slightly tighter for headings.

Paragraphs should not exceed comfortable line lengths (around 60–80 characters).

Typography should guide users’ eyes downward naturally, creating a visual rhythm that feels deliberate and balanced.

📱 Responsiveness

All text elements should scale fluidly between mobile and desktop.

Use proportional adjustments so headings remain visually strong but never overwhelm smaller screens.

Maintain readability — no excessively small fonts on mobile and no oversized ones on desktop.

Ensure line lengths and spacing adapt to preserve visual harmony at all screen sizes.

🎨 Visual Consistency

Apply consistent font weights and styles across the same text types (e.g., all section titles should share identical font weight).

Keep color usage uniform — main text color for content, muted tones for metadata or secondary info.

Avoid mixing too many text colors or font weights; the design should feel unified and professional.

Ensure text contrast passes accessibility checks and remains easy to read on the dark background.

🧩 Component-Specific Typography Rules

1. Hero Section

The main headline should be large, bold, and commanding — the visual anchor of the page.

Subtext below should use Inter, slightly smaller with relaxed line spacing for a clear, friendly tone.

Maintain generous spacing between headline and subtext. The layout should feel open and inviting.

Avoid excessive words; let the typography and spacing create impact.

2. Navigation Bar

Use Inter with medium weight for all nav links to ensure legibility and modern feel.

Keep text casing consistent (preferably Title Case or Uppercase).

Maintain equal letter spacing between items and enough padding for touch-friendly design.

Navigation text should feel balanced against the logo — not too bold, not too light.

3. Buttons & CTAs

Use Inter in medium or semibold weight for all button text.

Text should be clear and centered, with enough padding to look approachable and clickable.

Maintain consistent capitalization style across all buttons (either all uppercase or title case).

Ensure CTAs contrast visually with surrounding text but remain aligned with the site’s font system.

4. Section Titles & Cards

Section titles should use Space Grotesk with strong presence, visually separating content blocks.

In cards (e.g., project showcases), the card title should use Space Grotesk with medium weight, while descriptive text uses Inter with lighter weight.

Maintain consistent spacing between the title, description, and any metadata (e.g., date, tags).

Typography within cards should look balanced, with enough white space to prevent clutter.

5. Footer

Footer text should be smaller and subtler — use Inter in lighter weights.

Ensure sufficient contrast for readability but keep the tone understated.

Maintain equal spacing between footer elements for a structured, calm layout.

✨ Overall Direction

Typography should feel like the invisible grid of the website — quietly shaping the design’s tone and flow.
Every text element should communicate intentionality, with hierarchy, spacing, and scale used to guide the viewer effortlessly.
The end result should evoke confidence, clarity, and technical sophistication, just like the developer’s craft itself.
